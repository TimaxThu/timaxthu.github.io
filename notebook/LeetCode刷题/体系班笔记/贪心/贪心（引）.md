# 一个完整的贪心流程

## 题目

给定一个由字符串组成的数组strs，
必须**把所有的字符串拼接起来**，
返回所有可能的拼接结果中，**字典序最小的结果**





## 贪心策略

【考虑的贪心策略1】

把字符串按照字典序从小到大排序，之后输出

```java
if (s1 < s2) {
    // s1放前面
} else {
    // s2放前面
}
```

<font color= red>这种贪心策略是错的</font>。反例：

s1 = "b"，s2 = "ba"
如果用上述贪心策略，返回的是"bba"，但是字典序最小的结果是"bab"



【贪心策略2】

```java
if (s1+s2 < s2+s1) {
    // s1放前面
} else {
    // s2放前面
}
```





## 证明

首先证明策略具**有传递性**

如果a拼接b≤b拼接a，b拼接c≤c拼接b，则a拼接c≤c拼接a

# 贪心算法求解的标准过程

1. 分析业务
2. 根据业务逻辑找到不同的贪心策略
3. 对于能举出反例的策略直接跳过，不能举出反例的策略要证明有效性

这往往是特别困难的，要求数学能力很高且不具有统一的技巧性



---



# 贪心算法的解题套路

1. 实现一个不依靠贪心策略的解法X，可以用最暴力的尝试
2. 脑补出贪心策略A、贪心策略B、贪心策略C...
3. 用解法X和对数器，用实验的方式得知哪个贪心策略正确
4. 不要去纠结贪心策略的证明
